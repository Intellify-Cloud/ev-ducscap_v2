# Duces Capital

Jekyll site for Duces Capital.

## Build

Install Ruby and Node dependencies first:

```powershell
bundle install
npm install
```

Build the webpack assets and generate the Jekyll `_site` folder:

```powershell
npm run build
```

Use `npm run build` for production deploys. It runs `npm run bundle:prod` first,
which regenerates `assets/bundle.css` and `assets/bundle.js`, then runs
`bundle exec jekyll build` to generate `_site`. Do not deploy output from
`jekyll serve`; it can emit localhost absolute URLs in SEO metadata.

For local development:

```powershell
npm run serve
```

The `_site` folder is generated by Jekyll and intentionally ignored by git. The npm scripts run Jekyll through Bundler so the locked Ruby gem versions are used.

## Site architecture

- The main navbar is driven by `_data/navigation.yml`.
- `_includes/navbar.html` reads `site.data.navigation.cta` for the primary
  button and loops through `site.data.navigation.nav` for the hamburger menu.
- Add, remove, rename, or reorder main navigation links in the YAML data file,
  not in the navbar include.
- The blog index is `blog.html`, not Markdown. Keep HTML-heavy pages that use
  Liquid includes as `.html` files so Kramdown does not wrap include output and
  disturb layout/CSS behavior.
